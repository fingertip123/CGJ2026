# CGJ2026

Godot 2D space route prototype.

## 项目定位

本项目目标是一款 2D 太空航线规划游戏。主角是一艘飞船，需要在燃料有限的情况下到达锚地。飞船无法直接飞到终点，玩家需要规划初始航线，并利用中途星球的引力弹弓改变轨道，最终抵达锚地。

关卡路径上会存在星球防御塔或敌对单位。玩家需要在路线规划与飞船防御之间做取舍：既要让飞船进入正确轨道，也要避免飞船在途中被摧毁。

## 目标玩法

- 飞船初始位置固定，玩家不能通过绘制航线改变起飞点。
- 玩家点击方向点来设定飞船的初始航线方向，而不是绘制曲线。
- 方向设定后，玩家可以拖拽方向点来调整飞船的初始飞行方向。
- 设置方向时会显示直线轨道预览，帮助玩家判断发射后的飞行趋势。
- 飞船沿玩家设定的初始方向起飞，燃料有限，不能简单直飞终点。
- 路径上的星球提供引力弹弓效果，飞船需要借助引力完成轨道修正。
- 玩家可消耗资源给飞船加装防御模块或护航单位。
- 路径上的敌方防御会攻击飞船，飞船需要在生存和抵达锚地之间取得平衡。

## 当前实现状态

当前版本是一个可运行的原型，已经具备核心框架和基础路径编辑能力，但还没有完整实现引力弹弓。

已实现内容：

- 主场景 `scenes/Main.tscn`，入口配置在 `project.godot`。
- 1024x600 的 2D 游戏窗口布局，上方为飞行区域，底部为卡池 UI。
- `Background` 节点负责绘制飞行区域背景和网格。
- `RouteManager` 使用飞船当前位置作为固定起点，支持玩家点击方向点设定初始飞行方向。
- 方向设定后，可以拖拽方向点调整航线方向，并显示虚线直线轨道预览。
- `AnchorPoint` 是固定锚地，不会随玩家绘制的路径终点移动。
- 飞船节点沿设定方向飞行，进入固定锚地范围后胜利。
- 飞船拥有生命值、锚定范围、基础攻击能力和升级等级。
- 敌人会在飞行阶段从航线两侧生成，并尝试攻击飞船或护航无人机。
- 玩家可以通过底部卡池消耗资源部署护航无人机。
- 护航无人机会围绕飞船移动，并在飞船锚定范围内拦截敌人。
- 飞船升级会提升生命值、锚定范围、卡池槽位和无人机上限。
- 支持发射、重置、卡池刷新、飞船升级等基础操作。

尚未实现或仍需替换的内容：

- 星球、引力场和引力弹弓物理逻辑。
- 燃料消耗与燃料不足失败条件。
- 星球防御塔的正式关卡机制。
- 关卡编辑、胜负结算扩展、正式美术资源和音效。

## 操作方式

- `Space`：发射飞船。
- `R`：重置当前场景。
- 准备阶段在飞行区域点击方向点：设定飞船初始方向。
- 拖拽橙色方向点：调整飞船初始方向。
- 底部卡牌按钮：购买并部署护航无人机。
- `Refresh All`：消耗资源刷新无人机卡池。
- `Upgrade Ship`：消耗资源升级飞船。

## 代码结构

- `project.godot`：Godot 项目配置，主场景为 `res://scenes/Main.tscn`。
- `scenes/Main.tscn`：主场景，组织背景、航线、飞船、敌人生成、UI 等节点。
- `scripts/game.gd`：游戏主流程，负责阶段切换、资源、卡池、胜负和单位管理。
- `scripts/route_manager.gd`：航线方向设定、直线预览和路径采样。
- `scripts/player_ship.gd`：飞船主体逻辑，包含移动、生命、升级和攻击。
- `scripts/anchor_point.gd`：锚地点逻辑，负责锚地范围显示和停靠判定数据。
- `scripts/drone_escort.gd`：护航无人机逻辑，包含环绕、索敌、攻击和承伤。
- `scripts/monster.gd`：敌方单位逻辑。
- `scripts/spawn_manager.gd`：飞行阶段的敌人生成逻辑。
- `scripts/card_pool.gd`：底部卡池 UI 和按钮事件。
- `scripts/unit_data.gd`：飞船等级和无人机数据配置。
- `scripts/background.gd`：主场景背景和网格绘制。

## 开发环境

- Godot 3.x 项目。
- 当前渲染配置使用 GLES2。
- 当前项目已配置 Cursor Godot MCP，配置文件位于 `.cursor/mcp.json`。
